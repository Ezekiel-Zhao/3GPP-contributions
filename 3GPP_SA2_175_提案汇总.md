### 三、竞争焦点分析

1. **核心控制实体之争**: CMF vs CCCE vs CCAF — 谁来做计算与计算协调的决策者？
   - **CMF 阵营**（Huawei 联盟、OPPO、CATT、Lenovo、ZTE）：主张在 6G CN 内新增 CMF 实体
   - **CCCE 阵营**（ETRI、NTT DOCOMO、DT、AT&T）：主张 CCCE 执行 CC Policy，与 SHE Controller 交互
   - **CCAF 阵营**（Nokia、Ericsson、MediaTek）：主张 AF 侧决策，CN 仅提供暴露

2. **UE 请求路径之争**: SM NAS (Var A) vs Computing NAS (Var C) vs User Plane (Var E) vs New Plane (Var F)
   - 大多数公司支持 Var A（SM NAS），认为复用现有流程最务实
   - Huawei 联盟支持多路径并存
   - IIT Bombay 为 Var E 引入 SeMF

3. **计算节点地址传递方式**:
   - **NAS 传递**（OPPO、China Mobile）：CN 通过 NAS 将计算节点地址传递给 UE
   - **用户面路由**（ETRI）：通过 DNAI/UL CL/BP 路由，不暴露地址给 UE

4. **计算资源定义粒度**:
   - **粗粒度**（Nokia、NEC）：高层定义，留待后续细化
   - **中等粒度**（Ericsson、Samsung）：区分物理/虚拟/应用层
   - **细粒度**（Huawei 联盟、China Telecom）：定义具体资源类型、指标和三级暴露体系

| 提案编号 | 标题 | Summary |
|---------|------|---------|
| S2-2606356 | [KI#22, Solution Variant 22.10, update] Network metrics exposure to AF | 提出将预测性分析（NWDAF 派生）和网络可达性上下文暴露给 AF |
| S2-2606482 | [KI#22, Solution 22.8, update] Computing resource authorization by computing framework *(与 Nokia, LGE 联合)* | 描述 AF 或 UE 应用客户端的计算资源授权/请求流程（SA2 范围外） |

**详细分析 — S2-2606356**:
本提案对 Solution 22.10 进行了重要更新，核心贡献在于引入了两类新的网络信息暴露能力：一是基于 NWDAF 的预测性分析数据，包括预测路径时延、QoS 可持续性、拥塞预测和 UE 轨迹预测；二是网络可达性上下文，包括 DNAI 关联和可达性指示。提案引入 Correlation ID 作为 AF 与 6G CN 之间的不透明共享引用标识，使得 AF 可以在不暴露内部网络拓扑的前提下获取有价值的网络状态预测信息。这一设计对 6G 计算架构中 AF 侧的智能调度决策具有重要意义，使应用层能够提前感知网络变化并优化计算任务的分配策略。提案标题也相应修订为"Network metrics exposure to AF"以更准确地反映其范围。
